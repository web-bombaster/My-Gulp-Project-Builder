# Gulp-Builder
Предназначен для верстки макетов сайтов. Для удобной верстки с возможностью инклюда блоков на страницу и прочих плюшек, в проекте используется Pug.

Он подходит для:
- верстки лендингов
- многостраничных сайтов
- подготовки к интеграции в CMS
- мелких production-проектов

## Основная идея
Один сборщик у всех проектов, а не свой сборщик у каждого проекта. Экономим кучу места на диске. Сама gulp-сборка может лежать в любом месте (не зависит от нахождения проектов).

## Особенности
✔ Режимы дев и прод
✔ Watch режим (отслеживание изменений)
✔ Лайв-сервер (моментальное отображение изменений)
✔ Компиляция Pug
✔ Компиляция Sass
✔ Autoprefixer
✔ Sourcemaps
✔ Инклюды JS
✔ Для картинок делается webp с hash. В html делается подмена картинки на webp (в pug пишем img(src="/assets/images/css.png" alt="") , а в html получаем для img "/assets/images/css-e9f539abd7.webp")
✔ Шрифты - сканирует файлы шрифтов. На их основе автоматически генерирует @font-face в src/sass/base/_fonts.scss

## Как запускать сборку
1) Перед запуском сборки в gulpfile.cjs указываем: 
	- в `root` - путь до папки со всеми проектами, внутри которой будет находиться текущий проект (Пример: const root = 'D:/projects/';).

2) В терминале переходим в папку сборщика.
	- gulp --project=projectFolder - дев режим
	- gulp --project=projectFolder --prod - прод режим
	- gulp clean --project=gorman-main2 - очистить dist
В командах выше projectFolder - имя папки текущего проекта.

## Про режимы сборки:

### DEV режим (без --prod)
HTML форматируется (читаемый код) - .pipe(pug({ pretty: true }))
CSS - нет sourcemaps, включён cleanCSS, файл минифицирован
JS - код не минифицируется, просто сборка include-файлов
Images - картинки из src копируются в dist, без webp, без hash
Сервер - запускается BrowserSync, работает watcher, всё пересобирается при изменениях

### PROD режим (--prod)
HTML - Pug компилируется без pretty, пути картинок меняются на webp, подставляет хэш
CSS - sourcemaps включены, нет минификации, код читаемый
JS - код минифицирован
Images - через sharp конвертируются в .webp, через rev() получают hash
Сервер - server запускается, Watcher не запускеются.

## Рекомендуемая структура проекра
Под указанную ниже структуру настроена сборка. Но вы можете переопределить конфиг под ваши нужды.
Начальную заготовку проекта можно посмотреть тут:

```text
├─ dist/    ← сюда всё собирается при билде
├─ src/
│   ├─ assets/
│   │   ├─ fonts/    ← закидываем сразу woff2 шрифты
│   │   │   └─ ...
│   │   ├─ icons/
│   │   │   └─ ...
│   │   ├─ images/
│   │   │   └─ ...
│   │   └─ libs/    ← для подключаемых библиотек
│   │       └─ ...
│   ├─ js/
│   │   ├─ myscripts
│   │   │   ├─ helpers.js
│   │   │   └─ ...
│   │   └─ common.js    ← сюда инклюдим скрипты из myscripts
│   ├─ libs/
│   │   └─ libs-includ.js    ← сюда инклюдим скрипты библиотек из src/assets/libs/
│   ├─ pug/
│   │   ├─ bloks/    ← сюда добавляем блоки
│   │   │   ├─ head.pug
│   │   │   ├─ header.pug
│   │   │   ├─ footer.pug
│   │   │   └─ ...
│   │   ├─ components/    ← сюда добавляем компоненты
│   │   │   ├─ button.pug
│   │   │   └─ ...
│   │   └─ pages/    ← сюда добавляем страницы
│   │       ├─ index.pug
│   │       ├─ about.pug
│   │       └─ ...
│   └─ scss/
│       ├─ base/    ← базовые стили (не блоки и не компоненты)
│       │   ├─ _fonts.scss
│       │   ├─ _main.scss
│       │   ├─ _normalize.scss
│       │   ├─ _vars.scss
│       │   ├─ _verstka.scss    ← стили для страницы-меню для показа верстки
│       │   └─ ...
│       ├─ blocks/    ← блоки
│       │   ├─ _footer.scss
│       │   └─ ...
│       ├─ components/    ← компоненты
│       │   ├─ _burger.scss
│       │   ├─ _button.scss
│       │   └─ ...
│       └─ main.scss    ← тут делаем импорты всех других scss
├─ .gitignore
└─ README.md
```
